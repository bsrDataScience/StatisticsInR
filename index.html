k-nearest neighbors

k-nearest neighbors (KNN) is a machine learning algorithm that utilizes a non-parametric classification method (commonly measurement comparison by Euclidean distance) and a plurality among neighbors to make a prediction. k denotes the amount of neighbors being considered and must be selected carefully as prediction accuracy is sensitive to this parameter. There is no single best approach for selecting an optimal k, however, it is common practice to start with the square root of n number of observations used for training. In classification, ties are broken randomly, in regression, the predicted value is determined by an average of k selected nearest neighbors.

I. Classification

Our aim with classification is to predict a label. We will demonstrate classification with KNN twice, once with Edgar Andersons iris data and again with Iain Murrays fruits with colors data. From iris, we will predict species by sepal and petal dimensions: 4 total predictors. Then, from fruits with colors, we will predict the type of fruit using 4 predictors: width, height, mass, and color score.

B. Prepare the iris data for the KNN algorithm

Distance based algorithms are sensitive to the scale of the variables involved. Without normalization or scaling, the algorithm will be biased toward variables of greater magnitudes. We will be using min-max scaling, which will scale all of our data between 0 and 1, giving each feature equal importance.

    1. Normalization function
normalize <- function(x) {
  return((x - min(x)) / (max(x) - min(x)))
}

Now, we will subset the data, using 75% of the original observations (randomly organized) to train KNN, and reserving the other 25% for testing. Note that the training and testing subsets are each broken into two parts: data and labels.

    3. Create training and testing data sets

C. Apply the KNN algorithm

    1. k-nearest neighbors with iris - using the knn() function from the 
    CLASS package. We will keep k odd to mitigate randomness introduced 
    from tie-breaking. A confusion matrix is then created with the observed 
    labels of the testing values compared against the KNN predicted labels:
library(class) # for use of the knn() function
k1 <- round(sqrt(nrow(trainingData)))
if ((k1 %% 2) == 0) {k1 <- k1 + 1} # make sure k is odd
irisPrediction <- knn(train=trainingData, # training data
                      test=testingData, # testing data
                      cl=trainingLabels, # training labels
                      k=k1) # how many neighbors
pander(table(testingLabels,irisPrediction)) # confusion matrix
setosa versicolor virginica
setosa 15 0 0
versicolor 0 12 0
virginica 0 1 10

We only had 1 misfire: we predicted versicolor when it was in fact a virginica. Otherwise, very strong results from the algorithm: 37/38

    2. Check accuracy with the confusionMatrix() function from the caret package
library(caret) # for use of confusionMatrix()
cf1 <- confusionMatrix(testingLabels,irisPrediction)
print(paste(signif(cf1$overall[1],digits=3)*100,"% accurate."))
## [1] "97.4 % accurate."

Conclusion

Classification with k-nearest neighbors was quite accurate. In both of our test cases using the iris data and the fruits with colors data, we achieved over 93% prediction accuracy. Often, it will be necessary to adjust the k value to achieve the best possible results as there is no guaranteed method for determining the optimal k. In our case, we did not have to fiddle with k to get satisfactory classification predictions.

Regression with k-nearest neighbors was serviceable, at the very least comparable to linear regression. In our test case with the Boston data, our most extreme KNN predictions were further away from the observed values than the fitted values from MLR. However, our overall mean difference in median home value between the predictions and observed values were still mostly similar. MLR was roughly 5.7% closer to the true observed values of the homes, thus KNN had 6% worse prediction accuracy (they differed by $199).

k-nearest neighbors appears to be a great tool for making rough predictions for both labels and numeric values. As a non-parametric statistical method, we do not need to satisfy assumptions typically required by statistical modeling techniques. Due to this fact, KNN seems well suited for real world data where a specific underlying distribution may not exist.
